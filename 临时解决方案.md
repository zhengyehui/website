# 临时解决方案 - 方法二

## ✅ 已应用：暂时返回空数组

我已经修改了所有使用 Contentful 的地方，暂时返回空数组。

### 修改的文件

1. **`app/page.tsx`**
   - 移除了 `getPosts()` 调用
   - 直接返回空数组：`const posts: Post[] = []`

2. **`app/news/page.tsx`**
   - 移除了 `getPosts()` 调用
   - 直接返回空数组

3. **`app/news/[slug]/page.tsx`**
   - 移除了 `getPosts()` 和 `getPostBySlug()` 调用
   - 直接返回 404

## 🚀 现在可以构建了

```bash
npm run build
```

应该可以成功编译了！

## 📝 等 Contentful 配置好后

当你在 Contentful 中配置好内容后，可以：

1. 取消注释相关代码
2. 恢复 `getPosts()` 调用

例如在 `app/page.tsx` 中：

```typescript
// 取消注释这两行
import { getPosts } from '@/lib/contentful';
const posts: Post[] = await getPosts(3);

// 删除或注释这行
// const posts: Post[] = [];
```
