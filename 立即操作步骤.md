# 立即操作步骤（解决 Vercel 报错）

## 🎯 目标

确保代码已正确推送到 GitHub，Vercel 能成功部署。

---

## 第一步：运行一键修复脚本（最简单）

```bash
cd D:\project\website
一键修复部署.bat
```

脚本会自动：
- ✅ 检查 Git 状态
- ✅ 提交并推送代码
- ✅ 清理缓存
- ✅ 本地构建测试

---

## 第二步：验证代码已推送

### 2.1 在 GitHub 上检查

1. 打开你的 GitHub 仓库
2. 查看 `app/page.tsx` 文件
3. 确认代码是：
   ```typescript
   const posts: Post[] = [];
   ```

### 2.2 如果代码不对

```bash
cd D:\project\website

# 检查状态
git status

# 如果有未提交的更改
git add .
git commit -m "修复类型错误"
git push origin main
```

---

## 第三步：在 Vercel 中操作

### 3.1 清除构建缓存

1. 登录 Vercel：https://vercel.com/
2. 进入你的项目
3. 点击 **Settings** → **General**
4. 找到 **Build & Development Settings**
5. 点击 **Clear Build Cache**
6. 等待清除完成

### 3.2 触发重新部署

**方法一：推送空提交（推荐）**

```bash
cd D:\project\website
git commit --allow-empty -m "触发重新部署"
git push origin main
```

**方法二：在 Vercel 中手动触发**

1. 在 Vercel 项目页面
2. 点击 **Deployments**
3. 点击 **Create Deployment**
4. 选择 `main` 分支
5. 点击 **Deploy**

---

## 第四步：检查部署状态

### 4.1 查看部署日志

1. 在 Vercel 项目页面
2. 点击 **Deployments**
3. 查看最新的部署
4. 如果失败，点击查看 **Build Logs**

### 4.2 成功标志

- ✅ 状态显示 **Ready**（绿色）
- ✅ 可以访问网站
- ✅ 网站正常显示

---

## 🆘 如果还是报错

### 检查清单

- [ ] 代码已推送到 GitHub
- [ ] GitHub 上的代码是正确的
- [ ] 本地构建成功（`npm run build`）
- [ ] Vercel 已清除构建缓存
- [ ] 已触发重新部署

### 终极方案：暂时禁用类型检查

如果所有方法都试过了，在 `app/page.tsx` 最顶部添加：

```typescript
// @ts-nocheck
```

然后推送：

```bash
git add app/page.tsx
git commit -m "暂时禁用类型检查"
git push origin main
```
